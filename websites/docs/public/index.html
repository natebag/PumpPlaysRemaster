Quick Start

1

Go to the PPP Token Page

Visit pump.fun/coin/DxKwgDV2NZapgrpdvCHNdWcAByWuXvohKsdUAxcrpump and open the chat.

2

Type a Command

Send a message starting with a command prefix (-, /, !, or .) followed by a button name. Example: -a, -up, !start

3

Watch Democracy in Action

Every 3 seconds, all votes are tallied. The most-voted command wins and gets executed in the game. Your vote always counts!

4

Earn Rewards

Active players earn points that appear on the leaderboard. Top performers receive PPP tokens distributed hourly.

How Voting Works

The voting system runs on a 3-second cycle. Here's exactly what happens:

COLLECTING 0 – 3s

All commands sent in chat during this window are counted as votes. Each user gets one vote per cycle. If you change your mind, your new command replaces your previous vote.

PROCESSING instant

Votes are locked. The system tallies all votes and determines the winner. No new votes accepted during this phase.

EXECUTING instant

The winning command is sent to the emulator and executed in the game. All voters who picked the winning command earn bonus points.

Anti-Spam: Each user can only vote once per cycle. Rapid duplicate messages are ignored. Only valid game commands are accepted—pure text spam is filtered out early.

Advanced Input

Hold Commands

Some games support holding a button for a duration instead of a quick press. Use the hold prefix:

-hold right 2000 → Hold Right for 2 seconds
-hold a 500 → Hold A for 500ms
-hold up → Hold Up for default duration (1500ms)

Duration range: 100ms to 5000ms. Default if omitted: 1500ms.

Momentum Commands

Chain multiple presses of the same direction to move further:

-up3 → Press Up 3 times in sequence
-right5 → Press Right 5 times in sequence
-a2 → Press A twice

Stadium Team Prefixes

During Pokemon Stadium 2 (4-player mode), prefix your command with a team number:

-1a → Team 1 presses A
-2b → Team 2 presses B
-3up → Team 3 presses Up
-4start → Team 4 presses Start

If no team prefix is given, the command goes to the default player (Team 1).

Send PPP tokens to a burn address to inject commands directly into the game, bypassing the democratic voting system. Burns are verified on-chain in real-time. Higher burn amounts can queue multiple commands.

Leaderboard & Rewards

How Points Work

  • Vote submitted: +1 point for participating in a vote cycle
  • Winning vote: +2 bonus points if your command wins the tally
  • Combo bonus: Points multiplied for coordinated command sequences
  • Streak bonus: Consecutive winning votes earn escalating bonuses

Rankings

The leaderboard tracks multiple timeframes:

  • Hourly: Resets every hour. Top hourly players receive PPP distributions.
  • All-Time: Cumulative totals. Your legacy on the platform.

Reward Distribution

PPP tokens are distributed hourly to top leaderboard performers. The exact amounts depend on the reward pool configuration and number of active players.

Team Rocket

The Team Rocket terminal lets you bypass democracy by burning PPP tokens.

How It Works

  1. Connect your Solana wallet (Phantom recommended)
  2. Select a command to inject
  3. Burn the required amount of PPP tokens
  4. The system verifies the burn on-chain
  5. Your command is injected directly into the game, bypassing the vote
Warning: Burns are permanent and irreversible. The tokens are sent to a dead wallet and cannot be recovered. Use wisely.

Champions DAO

Champions DAO gives VIP access to top PPP holders without requiring token burns.

Requirements

  • Minimum Balance: 5,000,000 PPP tokens in your connected wallet
  • Verification: On-chain balance check via Solana RPC
  • Benefits: Free command injection, boosted vote weight, exclusive access

Unlike Team Rocket, Champions DAO doesn't destroy tokens. As long as you hold the minimum balance, you retain access. Sell below the threshold and access is revoked automatically.

Predictions

Prediction markets let the community bet PPP on game outcomes. Will we beat the gym leader? Will we catch the legendary? Bet on it.

How Predictions Work

  • Predictions can be created for any game event
  • Players bet PPP on their predicted outcome
  • Some predictions auto-resolve via game state reading (RAM)
  • Winners split the pool proportionally to their bet size

Bounties

Community-funded challenges with PPP prize pools. Anyone can create a bounty, anyone can contribute to the pool, and the first to complete it claims the reward.

Example Bounties

  • "Beat Brock without taking damage" — 50,000 PPP pool
  • "Catch a shiny Pokemon" — 100,000 PPP pool
  • "Win a Stadium tournament without losing a round" — 200,000 PPP pool

Combos

When the community coordinates a specific sequence of commands in a row, it triggers a combo bonus. Combo points multiply the standard point reward.

The system automatically detects coordinated sequences and rewards all participants who contributed to the combo chain.

Auto-Highlights

The system monitors game state via RAM reading and automatically detects key moments:

  • Pokemon caught or evolved
  • Gym badge earned
  • Party member fainted
  • Elite Four battles
  • Rare encounters

High-priority events trigger the overlay to display a highlight banner and can trigger clip saves for stream highlights.

Architecture

PUMP PLAYS REMASTER is built as a single Node.js application with a modular, event-driven architecture.

pump.fun Chat → ChatManager → CommandParser → VoteManager ↓ EmulatorManager / | \ BizHawk PJ64 Dolphin (Lua) (ViGEm) (ViGEm) ↓ GameStateReader ↓ Overlay / Leaderboard / Economy

Key Design Decisions

  • Config-driven: Adding a new game = adding a JSON config file. No code changes.
  • EventBus: All modules communicate via events, enabling loose coupling.
  • HTTP Polling for BizHawk: Lua polls the Node.js server for commands (no file-based IPC).
  • Command ID/ACK: Every command gets a unique ID. BizHawk ACKs after execution. No commands lost.
  • SQLite: All stats and leaderboard data in SQLite, not fragile JSON files.
  • Feature Flags: Non-MVP features are flagged off until stable.

API Reference

The backend exposes a REST API on the configured port (default 4000).

Status

GET /api/status

Returns current game, chat status, emulator status, vote stats, and enabled features.

GET /api/stats

Returns total users, total commands, and aggregate statistics.

Voting

GET /api/votes

Returns current vote cycle stats (active votes, phase, time remaining).

GET /api/votes/history

Returns recent vote results (winning commands, vote counts).

Games & Schedule

GET /api/games

Lists all configured games with ID, name, and system.

GET /api/schedule

Returns the weekly schedule and current day's entry.

GET /api/commands

Returns valid commands for the currently active game.

Leaderboard

GET /api/leaderboard?limit=20&order=total_points

Returns all-time leaderboard rankings.

GET /api/leaderboard/hourly

Returns current hour's leaderboard.

GET /api/user/:userKey

Returns a user's full profile (stats, rank, achievements).

Economy

POST /api/wallet/register

Register a Solana wallet address. Body: { userKey, walletAddress }

GET /api/rewards/pending/:userKey

Returns pending PPP rewards for a user.

Game Control

POST /api/command

Submit a command as a vote. Body: { command, username }

POST /api/command/direct

Direct command injection (bypasses voting). Body: { command, team }
